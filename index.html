The Project: FreshLens

One AI feature, built for real and shipped to the Expired Solutions app.

Today the app scans one item at a time. FreshLens is the 10× version: point your camera at a whole fridge, basket, or produce shelf and every item lights up at once with a live, color-coded freshness score. Then AI turns that into action: for shoppers, an “eat me first” ranking and a recipe to rescue what’s about to turn; for store managers, an instant markdown worklist and projected shrink savings.

1

Multi-Item Detection

A pretrained object detector finds every produce item in the frame and draws a box around each one—turning a single-item scanner into a whole-scene scanner.

2

Live Freshness Overlay

Each detected item is scored 0–100 by the freshness model and shown with a color-coded overlay (green, amber, red) right on the camera view.

3

Act On It

Claude turns scores into action: an “eat me first” ranking and a recipe for shoppers, or a markdown worklist and shrink-savings estimate for store managers.

Get started — do this on Monday

This week the whole team builds FreshLens in one shared repo. It already runs out of the box — your job is to get it running, claim a slice, and ship your first pull request the AI-native way. Everything below is in the starter repo.

  1. Clone & set up — about 15 minutes. One command makes the virtual environment, installs everything, arms the safety guardrails, and runs the tests. git clone https://github.com/LawrenceHua/es-intern-freshlens.git cd es-intern-freshlens ./setup.sh # venv + deps + Claude Code hooks + self-test
  2. Run it and look around. uvicorn app:app --reload # then open http://127.0.0.1:8000 In Claude Code, run /onboard and read GETTING_STARTED.md. New to Claude Code? Read docs/USING-CLAUDE-CODE.md first.
  3. Claim a starter task. Pick one good first issue (one per slice: detect · freshness · act · frontend · report · eval, plus a Week-7 provenance stretch), then claim it — this also creates your branch: gh issue develop -c <issue-number>
  4. Build with AI, then ship. Work only in your files. Explore → Plan → Implement → run /code-review → open a pull request. CI (lint, tests, secret-scan) must pass and one teammate approves before it merges. Never push straight to main.
3 non-negotiables: keep the API shape {"status":"ok","data":…} · never commit secrets or a .env file · stay in your lane and leave pytest green.
