Flagship Project · Weeks 5–8 build

FreshProof

Verified digital receipts and item-level provenance for the Expired Solutions shopper app. Prove where and when every item was bought—the foundation for freshness guarantees and one-tap spoilage refunds.

Python FastAPI OCR SQLite Claude API Signed tokens JavaScript

The problem

When produce spoils days before it should, a shopper has no easy way to prove where or when they bought it—so they quietly eat the loss, lose trust, and stop buying fresh. Stores, in turn, have no clean signal to honor a guarantee. The missing piece is trustworthy provenance: a verifiable record that ties each item to the store, date, and price it came from.

FreshProof turns an ordinary receipt into that record. It lives at the moment right after purchase—a part of the shopper journey nothing else in the platform covers.

How it works

Three capabilities, built in sequence over the 4-week sprint.

1

Receipt Review

The shopper snaps or uploads a receipt. OCR parses every line item; the shopper confirms or corrects, and each item is matched to the produce knowledge base for an estimated shelf life.

2

FreshTag

Each confirmed item gets a virtual tag—a cryptographically signed record of item, store, date, and price—that flows into the shopper's inventory and powers expiry reminders.

3

Proof of Provenance

Any FreshTag can be verified: a valid signature proves the item genuinely came from that store at that time. Tampered or forged tags fail the check—the basis for refunds.

Architecture

A small FastAPI service over SQLite. The "proof" is a signed token (HMAC over the item record with a per-store key)—no blockchain required, fully verifiable, and teachable in four weeks.

4-week build plan

Maps to program Weeks 5–8. Each Friday ends with a live demo.

Week 5 · Receipt → data
  • SQLite schema: receipts, items, fresh_tags
  • Image upload endpoint (HEIC/JPEG/PNG)
  • OCR parsing of line items
  • Match items to the produce knowledge base
Demo: upload a receipt → see parsed items with estimated shelf life.
Week 6 · FreshTag + review UI
  • Sign a FreshTag per confirmed item
  • Review screen: confirm / correct parsed items
  • Persist tags; show them on each item
  • Mobile-first responsive layout
Demo: confirm a receipt → every item gets a FreshTag in the review screen.
Week 7 · Provenance verification
  • Verify endpoint: validate a tag's signature
  • Tamper detection (altered store/date/price fails)
  • Proof view: "Genuine · Store X · June 2"
  • Shareable proof link or QR
Demo: verify a tag → "Genuine"; edit one field → "Invalid / tampered".
Week 8 · Integrate + harden
  • Tags flow into inventory + expiry reminders
  • Refund-eligibility check (spoiled before shelf life)
  • Error handling + 10+ unit tests
  • Docs: API contract + setup
Demo: spoiled item → "Eligible for refund" with verifiable proof attached.
